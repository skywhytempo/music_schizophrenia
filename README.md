# Музыкальный анализатор вкуса

Небольшой питоновский проект, который тащит плейлист, кодирует его векторами (one-hot encoding) и пытается понять, из скольких «субличностей» состоит музыкальный вкус.

## Что делает проект

- Подтягивает треки из плейлиста (артист, название, жанр).
- Группирует жанры в укрупнённые кланы (rock/metal, rap, pop, electronic и т.п.).
- Превращает треки в one-hot векторы по артистам и жанрам.
- Строит «вектор среднего вкуса» и считает:
  - косинусное сходство каждого трека с этим вектором;
  - выборочное среднее и дисперсию сходства;
  - «индекс разнообразия вкуса».
- Проводит жанровой разбор:
  - считает статистику внутри каждого жанра;
  - показывает самых типичных треков и артистов;
  - рисует гистограммы распределения схожести.

Проще говоря, отвечает на вопрос: «Насколько ты музыкально стабилен, и где живут твои guilty pleasure?».

## Как запустить

python ohe.py

Затем:

1. Вставить ссылку на плейлист (поддерживается сервис, который парсит модуль `yme.py`).
2. Подождать, пока проект:
   - заберёт треки;
   - построит OHE;
   - посчитает статистику;
   - выведет списки треков и откроет гистограммы.

## Структура проекта

- `ohe.py` — точка входа: диалог с пользователем, запуск анализа.
- `yme.py` — работа с внешним сервисом: парсинг плейлиста, нормализация жанров.
- `analysis.py` — вся математика:
  - построение OHE;
  - центроиды;
  - косинусы, средние, дисперсии;
  - жанровой разбор.
- `view.py` — вывод результатов и графики (таблички в консоль, гистограммы).

## Идеи для развития

- Добавить кластеризацию треков (k-means) и назвать кластеры:
  - «ночной депрессивный рок»,
  - «утренний бодрый поп» и т.п.
- Сравнивать два плейлиста между собой:
  - «совместимость вкусов» друзей.
- Экспортировать результаты в HTML или Telegram-бота.
