# Музыкальный анализатор вкуса

Небольшой проект на Python, который вытаскивает плейлист с внешнего сервиса, кодирует его векторами (one-hot encoding) и пытается разобрать твой музыкальный вкус: от глобального «ДНК вкуса» до отдельных «эпох» плейлиста и поджанров.

## Что умеет сейчас

### 1. Анализ одного плейлиста

- Подтягивает треки (исполнители, названия, **сырой жанр** и **укрупнённый жанр**).
- Строит one-hot векторы по артистам и жанрам.
- Формирует «вектор среднего вкуса» (центроид).
- Считает:
  - косинусное сходство каждого трека с центроидом;
  - выборочное среднее и дисперсию сходства;
  - метрику «индекс разнообразия вкуса».

Выводит:

- топ‑N треков, **наиболее** и **наименее** похожих на твой «средний» вкус;
- «расшифровку ДНК вкуса» — доминирующие жанры и основные исполнители.

### 2. Жанровые кластеры и поджанры

Для каждого укрупнённого жанра (например, `rock_metal`, `electronic_dance`):

- выделяет треки этого жанра;
- считает центроид жанрового кластера и распределение схожести внутри него;
- находит:
  - топ‑артистов жанра;
  - самые «типичные» треки;
- анализирует **поджанры** внутри кластера по сырым жанровым меткам (например, как внутри `rock_metal` распределяются `rusrock`, `punk`, `numetal` и т.д.).

### 3. Эпохи плейлиста

Разбивает плейлист на несколько «эр» по порядку добавления треков (новые в начале, старые в конце) и для каждой:

- считает частоту жанров;
- выделяет основных артистов;
- делает простой «анализ настроения» по названиям треков:
  - считает, сколько «агрессивных» и «грустных» ключевых слов встречается (по заранее заданным спискам).

Это даёт небольшой «биографический» срез: как менялся жанровый профиль и настроение плейлиста со временем.

### 4. Сравнение двух плейлистов

Есть отдельный режим, который:

- берёт два плейлиста;
- строит общий словарь артистов и жанров;
- кодирует оба плейлиста в одном пространстве признаков;
- считает:
  - косинусное сходство глобальных центроидов вкуса;
  - пересечение артистов и жанров (включая Jaccard‑коэффициенты);
- выводит список общих артистов и общих жанров.

Идеально, чтобы сравнить «мой плейлист vs плейлист друга» или «старый vs новый».

## Структура проекта

- `yme.py` — работа с внешним сервисом:
  - парсинг плейлиста;
  - нормализация жанров (сырые жанры → укрупнённые группы).
- `PlayListData.py` — класс-обёртка над данными плейлиста:
  - список треков;
  - карты артистов и жанров.
- `analysis.py` — чистые функции анализа:
  - построение OHE;
  - центроиды;
  - косинусы, статистика;
  - анализ жанров, поджанров, эр.
- `Analyzer.py` — класс `Analyzer`:
  - использует `PlayListData` и функции из `analysis` для:
    - глобального анализа вкуса;
    - жанровых кластеров;
    - анализа по эпохам.
- `comparison.py` (планируется/если уже есть) — сравнение двух плейлистов:
  - общее пространство признаков;
  - сходство вкусов и пересечения.
- `view.py` — вывод результатов:
  - текстовые отчёты в консоль;
  - гистограммы распределения схожести.

- `ohe.py` — точка входа с небольшим меню:
  - анализ одного плейлиста;
  - жанровые и временные отчёты;
  - сравнение двух плейлистов.

## Как запустить

`python ohe.py`

Далее:

1. Выбрать режим в меню.
2. Вставить ссылку на плейлист (поддерживается формат, который парсит `yme.py`).
3. Ждать, пока программа:
   - заберёт треки;
   - построит OHE;
   - посчитает метрики;
   - выведет отчёты и, при необходимости, нарисует гистограммы.

## Идеи для развития

- K-Means по трековым векторам:
  - автоматические кластеры вкуса поверх ручных жанров.
- 2D‑визуализация структуры вкуса (PCA/UMAP + scatter plot).
- Более умный «анализ настроения» по названиям треков.
- Отдельный режим «рекомендаций внутри плейлиста»:
  - поиск похожих треков (k‑NN по вектору вкуса).
- Интеграция с API других стриминговых сервисов (как источник дополнительных фич вроде популярности/даты релиза).
